# 系统设计规则指南

## 文档说明

### 文档介绍

本文档归纳并约定项目在**系统设计**中使用的图示与规则，用于统一架构表达方式、提升文档可读性和团队协作一致性。所有与架构、模块划分、分层相关的文档与图示，建议遵循本指南中的规则。

### 规则的规则

本指南中每条规则按以下结构组织：

| 项 | 说明 |
|----|------|
| **使用范围** | 该规则适用于哪些场景、文档或角色（如：架构图、技术方案、全员）。 |
| **简介** | 规则的核心内容与目的（一两句话）。 |
| **示例** | 符合该规则的具体示例或反例说明。 |

若某条规则有额外约定（如格式、工具、禁止事项），会在该规则下单独列出。

## 一、系统模块结构图

### 使用范围

- 适用于：整体架构概览、分层架构说明、系统/子系统模块划分图。
- 适用文档：架构设计文档、技术方案、对外/对内技术介绍。
- 适用角色：架构师、开发、产品/技术 PM。

### 简介

**系统模块结构图**用于表达系统的分层与模块划分：按**层级**自上而下排列，同一层内可有**多个模块组**并列；同一分组内的**模块横向排列**；层名可置于左侧作为标注。不表达模块间的调用/数据流细节时，可不画连线，仅通过分层与分组表达结构。

### 绘图规则

#### 布局规则

1. **分层**：自上而下排列，上层偏业务，下层偏平台
2. **左侧层名**：每层最左一列标注层名，黑色文字 `#111111`，无填充
3. **同层多组**：同层允许并列多个分组
4. **分组名**：每个分组框内右上角标注分组名称
5. **组内排列**：组内横向等距，模块数量与标签长度尽量一致
6. **层间关系**：仅结构时可无连线，需要表达关系时再加纵向连线
7. **宽度**：各层/各组整体宽度保持一致或接近

#### 配色规则

8. **统一边框**：所有边框统一使用 `#94A8C4`
9. **层背景色**：
   - 层1：`#E3F2FD`（浅蓝）
   - 层2：`#E8F5E9`（浅绿）
   - 层3：`#FFF3E0`（浅橙）
   - 层4：`#F3E5F5`（浅紫）
   - 层5：`#E0F7FA`（浅青）
   - 层6：`#FCE4EC`（浅粉）
   - 层7：`#E8EAF6`（浅靛）
   - 层8：`#F1F8E9`（浅黄绿）

10. **模块填充色**：
    - 层1 模块：`#BBDEFB`
    - 层2 模块：`#C8E6C9`
    - 层3 模块：`#FFE0B2`
    - 层4 模块：`#E1BEE7`
    - 层5 模块：`#B2EBF2`
    - 层6 模块：`#F8BBD9`
    - 层7 模块：`#C5CAE9`
    - 层8 模块：`#DCEDC8`

11. **色彩原则**：层色与模块色一一对应，同层模块使用同一模块色，明度一致、色相区分
